保存游戏


描述

此动作将把游戏初步地存储进一个给定的文件。你需要提供需要加载文件的名称（字符串格式，包括扩展），然后此动作将会受限制地保存当前游戏。特别需要注意的是，此动作只是能够存储当前游戏状态的”简要说明“，因此它更像一个检查点系统（需要在固定的地方保存读取游戏），而不是全局的存储系统（随时随地保存游戏）。注意此动作 不保存任何动态资源 (比如 数据结构 或 粒子)。因此你需要特别小心地是用此函数来避免报错或存储泄露。为了更容易理解加载存储游戏数据的方法，请看 <a4>File Actions</a4>。
