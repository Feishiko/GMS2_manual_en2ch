数据结构（Data structures）

这一节包含了各种专门用于存储不同种类数据的函数

在游戏中，你经常需要精确且有序地存储信息。比如，你可能需要保存一个人携带的刀具列表或你希望存储一组需要访问的地区。那么，你 可以使用数组来操作，但是如果你想进行更复杂的操作，比如对数据进行排序或者搜索指定道具，你需要编写大量的GML代码，这些代码可能执行速度慢，难以调试，写起来十分痛苦
为了解决这个问题， GameMaker Studio 2有许多内置的 数据结构(Data Struactures) ，它们可以通过指定的函数进行访问调用。一共有六种不同类型的数据结构可供使用，每种数据结构都有其自身的优点，具体使用则取决于你希望存储的信息类型以及之后你需要对数据进行何种操作：堆栈、队列、列表、映射、优先队列以及栅格
基本上，所有的数据结构都以相同的方式工作——你创建数据结构并将其 索引值保存在一个变量中。然后，你可以使用这个索引值在其他函数中进行调用操作。最后，一旦你用完记得要销毁这个数据结构以将其从内存中移除。你可以根据需求来同事是使用多个数据结构，并且所有结构都可以存储字符串和数值

注意：与所有动态资源一样，数据结构会占用你的内存，因此在不再使用时 应当进行销毁，以防止内存泄漏导致游戏发生crash

关于不同数据结构的具体信息可以在以下章节找到：

  1. 堆栈（Stacks）
  2. 队列（Queues）
  3. 列表（Lists）
  4. 映射表（Maps）
  5. 优先队列（Priority Queues）
  6. 栅格（Grids）

在你实际使用数据结构之前，你还需要了解一点，在四舍五入的情况下有时候你得到的结果可能会与预期相悖。使用下列函数来修改数据精度可以解决该问题：

  • ds_set_precision

以下函数则用于检查某个数据结构是否存在

  • ds_exists
