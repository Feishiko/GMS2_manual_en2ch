文本文件

文本文件是在游戏外部存储大量数据的有用方法。 与 ini 文件相反，你最多可以同时打开 32 个文本文件，你也可以读取和写入大块数据。 但是，你可以决定如何构建数据，因为没有像 *.ini 文件 那样设置 “section（节）” 或 “key（键）” 结构。

请注意，对于具有本地化的游戏，从外部文件中读取重音字母非常重要。当之前没有提供文件时，你应 首先 以 UTF8 格式创建 txt 文件（而不是用 GameMaker Studio 2 文件函数创建的默认 ANSI 格式 txt 文件），然后将其作为内含文件添加到 GameMaker Studio 2 中，以便在运行游戏时导出并使用。通过这种方式，你可以正确地读取和写入所有重音和非罗马字母。

存在以下处理文件的函数：

  • file_text_open_read
  • file_text_open_write
  • file_text_open_append
  • file_text_open_from_string
  • file_text_read_real
  • file_text_read_string
  • file_text_readln
  • file_text_write_real
  • file_text_write_string
  • file_text_writeln
  • file_text_eoln
  • file_text_eof
  • file_text_close
警告！ 由于 GameMaker Studio 2 被沙箱化，这些函数可能无法正常工作！有关详细信息，请参阅 文件系统限制 部分。
