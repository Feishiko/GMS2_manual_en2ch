偏好设置 - 常规（General）


常规偏好设置 被用于定义GameMaker Studio 2 IDE的常规行为。最常规的几项已经被列在主页上，而其他的项目被分到多个的子目录下，我们会在后文深入解释。

  • 默认音频设备（Default Audio Device）：在此处你可以选择GameMaker Studio 2 IDE所使用的默认音频设备你可以通过点击刷新（Refresh）按钮来刷新设备列表。

  • 退出确认时默认（Default For Exit Confirmation）：该选项允许你设置在退出GameMaker Studio 2时是否显示确认信息。该选项默认为开（On），但你可以将其设置为允许放行该消息，就好像你已经点过确认按钮那样。

  • 自动重新载入变动文件（Automatically Reload Changed Files）：如果该选项被勾选，则IDE会自动扫描并检查

  • 立即打开新资源（Immediately Open New Resources）：如果此项被勾选，每次你在资源树创建新资源的时候，相应的资源编辑器会被自动打开。该选项默认被勾选，解除勾选该项意味着你需要手动打开所有新创建的资源。

  • 总是打开被拖拽到IDE中的新资源（Always Open New Resources Dropped Into IDE）：某些资源（像精灵（Sprites）或声音（Sounds））是可以被拖拽到GameMaker Studio 2主工作区中的，并且它能够被检测到并被添加到资源树中作为一个新资源。默认地，当这种情况发生时，相应的资源编辑器会被打开，而解除该选项将会阻止编辑器被自动打开，并且你需要手动地打开他们。

  • 允许通过快捷键关闭空标签（Allow Empty Tabs To Close Via Shortcut）：你可以使用键盘快捷键 + W关闭任意被打开的标签，但如果禁用该选项，则该快捷键将不再生效。该选项默认为启用

  • 禁用IDE切换效果（Disable IDE Transitions）：GameMaker Studio 2的IDE有一些动画切换效果。例如打开窗口等……然而这些效果并不是所有人的菜并且如果在老式或者低功耗机器上禁用这些效果你也可能得到性能的提升。你可以在此处禁用他们，它们默认是打开的。

  • 在启动时显示欢迎屏幕（Show welcome screen on startup）：该选项默认为开，并且只是要求GameMaker Studio 2在启动程序的时候显示一个简短的启动画面。解除此项会阻止该画面的显示

上述这些选项是在常规（General）偏好设置里可用的主要项目，但在下面列出的子目录中还有一些额外选项：

当你希望为一个物体添加精灵或者在拖拽按钮（Drag and Drop）中选用一个字体又或者是其他许多场合下，资源浏览器总会呈现到你的眼前：

默认地，它会同时显示一个树状视图（左侧）和一个列表视图（右侧），但通过使用该设定，你可以选择只显示一个或另一个。


背景偏好设置用于处理工作区的背景、显示什么背景以及背景如何去显示。选项有：

  • 背景图（Background Image）：该选项是被用于各个工作区下作为背景的图像文件。这里会提供一张默认图像，并且默认状态下是启用的，但是你也可以在这里选择你自己的图像。你可以选择的图像种类有*.png、*.gif、*.jpg、*.jpeg、*.tif、*.tiff、*.ico、*.bmp、*.2bp以及*.mdi，但是请注意，无论初始的文件格式如何，所有被使用的文件都会被另存为*.png文件，因此动画*.gif文件只会被渲染成一张单初始帧图像。

  • 恢复默认（Restore Default）：如果你改变了背景图而不是使用默认图像，你可以随时通过这个按钮来重置为默认图像

  • 图像尺度（Image Scaling）：该选项控制了背景图像在各个工作区下如何被缩放。选项有：

    • 拉伸（Stretch） - 图像将会被拉伸以适配工作区，拉伸尺度取决于可见的停靠栏。
    • 拼贴（Tile） - 图像将会被逐片拼贴到工作区上。
    • 最佳适配（Best Fit） - 图像将会被按比例地缩放来适配工作控件，如果不是1:1适配则会在边缘留白。
    • 桌面拉伸 - 该背景图将会覆盖整个工作区，无视所有可见的停靠栏。
  • 是否应该显示背景图（Should Background Images Be Shown）：该选项控制所选的背景图是否应该被显示。该选项默认为开（On）

编译器偏好设置用于处理某些在编译游戏以测试或创建程序包时和输出及编译处理的某些方面有关的选项。选项有：

  • 当错误发生时自动弹出语法错误（Automatically pop up Syntax Errors when an error occurs）：编译时启用或禁用语法错误通知。该选项默认为开

  • 在编译期间显示编译窗口（Show the compile window during compilation）：默认地，编译窗口会在你编译项目的时候被显示出来，即时在编译停靠栏已经关闭的时候（编译的时候再打开）。解除此选项则会关闭这个行为，而如果打开，编译窗口将会出现。

  • 编译时在编译窗口中显示所有编译指令（Show all compiler commands in compile window during compilation）：该选项默认被勾选，并强制编译器在编译窗口中显示出所有的相关指令。解除该选项会使得输出内容更简短，信息量更少。

  • 显示所有详细的编译指令（Show all verbose compiler commands）：勾选该项会强制编译器显示与编译进程相关的更多指令。一般地，这些内容只在调试的时候是需要的，并且在你遇到问题的时候可以请求支持。该选项默认为关（并且推荐为关）

  • 编译时清空输出窗口（Clear output window on compile）：默认地，编译窗口会在每次开始生成的时候清空窗口，而解除此项会保留之前的编译日志。该选项默认为开，并且这也是推荐选项，因为过多的编译器日志会拖慢编译时间。

  • 调试器使用的端口（Port for Debugger to use）：该选项设置调试模块所使用的网络端口。默认值为6509


确认对话框偏好设置用于处理一些在GameMaker Studio 2 IDE中执行某些动作时的常规弹窗确认信息。选项有：

  • 使用快捷键关闭非空工作区标签页时的默认处理（Default for closing non-empty workspace tab using keyboard shortcut）：前面所说的的键盘快捷键可以用来关闭任何打开的标签页，但有时候你可能会不小心关掉你所需要的标签页。为了防止这种情况，你可以将该选项设置为“显示消息”（Show message，也是默认选项）来询问你是否真的想要关掉那个标签页。另一个选项为“否”（No），如果快捷键被使用了，则可能什么也不做或者确认关闭标签页而不显示任何信息。

  • 重新运行生成项时的默认处理（Default for restarting a running build）：当你在上一个生成版本还在运行的时候你重新开始生成时，你会被询问是否想要先中止当前的生成版本。解除勾选此项会抑制该消息的产生并强制GameMaker Studio 2认为你已经点击“确认”（Okay）作为回应了。该选项默认为“显示消息”

  • 重置工程布局的默认选择（Default choice for resetting the project layout）当你（从布局菜单（Layouts Menu））选择重置工程布局时，默认地会显示一个警告消息告诉你该操作将会关闭所有打开的资源窗口。然而通过将此项选择为“是”（Okay），你可以抑制该消息的产生并自动重置布局，而如果选“否”（No）则什么也不会发生。默认值为“显示消息”

此处的偏好设置是用于前往（Go To）窗口的，你可以用键盘快捷键 + T打开它。

  • 搜索结果显示的最大数目（Maximum number of search results to display）：该选项控制着你使用前往搜索窗口时返回搜索结果的最大数量。

帮助文件被设计成向网页那样的操作方式并且因此需要使用GameMaker Studio 2的微型网络服务器（Micro Web Server，MWS），或者也可以使用外部浏览器。能够在这里被修改的设定有：

  • 网络服务器所使用的端口（Port number to use for the web server）：该选项设置了供MWS使用的端口号。默认值为51268

  • 使用外部浏览器打开帮助文件（Use external browser for help）：如果勾选此项，则以后会使用你系统上的默认浏览器打开帮助文件。该选项默认为开，关闭该选项则会强制帮助文件在IDE的工作区内打开。

输入偏好设置用于控制IDE中某些需要鼠标和键盘输入方面的控制。可用的选项有：

  • 慢双击时间（毫秒）（Slow double click time (ms)）IDE中的某些位置要求一次较慢的双击（慢双击，Slow Double Click）才能发挥作用，比如重命名一个资源（快速双击（快双击，Fast Double Click）会打开该资源的编辑器，而慢双击才允许你为它更名）。该选项允许你设置慢双击的侦测速度，默认值为350毫秒

  • 还原窗口所需要拖拽标签页的距离（Distance tab must be dragged to un-maximise the desktop）：当你将一个工作区标签从IDE中的默认位置拖拽出来时，如果与此同时只有一个活动工作区，则GameMaker Studio 2的窗口会被还原并跟随鼠标移动（如果你有多个活动工作区，则会创建一个新的IDE实例窗口来承载这个工作区）。该选项控制着在工作区识别出该操作之前你需要将标签页拖拽出多远，默认值为4px。

  • 锁定部件滚动状态的时间（Time in seconds to lock scrolling to a gadget）###此处原文给的是秒（Seconds），但IDE中给出的是毫秒（milliseconds），因此Seconds被略过###：该选项用于处理GameMaker Studio 2在两个交叠窗口之间滚动的等待时间。必要地，当你滚动窗口“A”时，光标可能会悬停在另一个可滚动的窗口“B”的部分区域上，因此这个设置定义了在你停止滚动“A”窗口之后需要多长时间你才能滚动“B”窗口，这样你就不会不小心地滚动到别的面板了。默认值为500毫秒

  • 允许鼠标与窗口内容互动的最小缩放级别（Minimum zoom level to allow mouse interaction with window contents）：该选项控制缩放的中断点，缩放级别低于该点时鼠标酱不再与工作区内的窗口进行互动（例如：增加节点、拖拽窗口等……）。默认值为40.

  • 必须按下Ctrl键来进行滚轮缩放（Control key must be held down for mouse wheel zoom）：当勾选此项时，所有编辑器以及工作区都必须使用+鼠标滚轮才能进行缩放，而解除此项则只需要鼠标滚轮就能进行缩放了。

输出窗口偏好设置控制着IDE中各个输出窗口的各种颜色和格式。每一个不同的窗口都共有相同的一组选项，列举如下：

  • 颜色（Colour）：文本被写入该窗口所使用的默认颜色。

  • 字体名称（Font Name）：所使用的字体名称（所可用的字体取决于安装在你系统上的字体）。

  • 字体样式（Style）：所使用的字体样式，比如粗体（Bold）或斜体（Italic）（请注意，不是所有窗口都会允许相同的字体样式，并且这也取决于所使用的字体）。

  • 大小（Size）: 用于所给定的输出窗口的字体大小。

  • 抗锯齿模式（Anti-Aliasing Mode）：在此处你可以设置输出窗口字体渲染的抗锯齿模式。你可以选择使用默认（Use Default）来让IDE决定，也可以通过选择相应选项来强制它开启（On）或关闭（Off）。

文件路径偏好设置（Path Preference）用于控制GameMaker Studio 2中的一些要素以及你所要求的外部编辑器（External Editors）所需的各种文件路径。在IDE中有如下选项：

  • 临时文件位置（Temp Location）：该位置用于存储所有的临时文件夹。默认地，该路径为\Users\<Username>\AppData\Local\Temp\GameMakerStudio2。

  • 资源缓存（Asset Cache）：该位置用于存储每个工程的资源缓存。默认地，该路径为D:\Users\<Username>\AppData\Roaming\GameMakerStudio2-Dev\Cache。

  • “我的工程”文件夹位置（"My Projects" Location）GameMaker Studio 2创建新工程的初始位置。默认地，该路径为D:\Users\<Username>\Documents\GameMakerStudio2。

  • 自动删除临时文件夹（Auto Delete Temp Folder）：勾选此项将会强制GameMaker Studio 2自动删除每次编译工程等时候产生的临时文件夹。该选项默认为开，解除勾选此项会将其关闭（意味着你需要在以后手动地删除所有的临时文件）。

  • 退出时删除资源缓存（Delete Asset Cache On Quit）：如果勾选此项，则在你退出GameMaker Studio 2时候删除资源编译器缓存。. 该选项默认为关，启用此项意味着每次你载入并运行工程时需要重新创建缓存（取决于游戏的大小，这可能会花一些时间）。

  • 删除临时文件夹（Delete Temp Folder）：点击该按钮将会删除该工程的临时文件夹。

  • 删除资源缓存（Delete Asset Cache）：点击该按钮将会删除该工程的资源缓存。

还有下列用于设定外部编辑器路径的选项：

  • SWF文件的外部编辑器/查看器（Path to external editor/viewer for SWF files）：如果你要处理SWF类型的精灵，你可以将该选项设置为你喜欢的查看器/编辑器的路径，并且当你在精灵编辑器（Sprite Editor）中点击编辑精灵（Edit Sprite）按钮的时候，它会打开被给定的程序。此处默认值为无路径。

  • Spine文件的外部编辑器/查看器（Path to external editor/viewer for Spine files）：如果你要处理Spine类型的精灵，你可以将该选项设置为你喜欢的查看器/编辑器的路径，并且当你在精灵编辑器（Sprite Editor）中点击编辑精灵（Edit Sprite）按钮的时候，它会打开被给定的程序。此处默认值为无路径。


节能偏好设置（Power Saving Preferences）将会影响生成GameMaker Studio 2 IDE所使用的CPU及GPU功率（必要时这些设置会在程序运行时降低基本帧率及功耗）。选项有：

  • % 在使用交流电源时（% when on AC power）：当连接输入电源时的基本功耗。默认值为100%

  • % 在使用电池电源时（% when on battery power）：当使用机器电池时的基本功耗。默认值为35%。

  • % 在应用程序失去焦点时（% when application is not in focus）：当GameMaker Studio 2失去焦点时的基本功耗（该情况下无视电源类型）。默认值为15%

此处的设置包含了某些资源该如何在工程中被绘制出来。

  • 在新工程中默认的绘制初始色（Initial default draw colour for new projects）：当你使用GameMaker Studio 2的函数或拖拽按钮（DnD™）动作调用字体或绘制形状时若没有指定绘制颜色，则会使用此处的颜色。默认颜色为白色($ffffff)。

此处的设置将影响到你能如何在GameMaker Studio 2各个工作区之间游动浏览。选项有：

  • 使用快捷键浏览窗口时允许的角度范围（The degree range allowed when looking for a window using shortcuts）在一个工作区内你可能会打开多个不同物体和资源的窗口，并且为了快速地在它们之间移动，你会使用键盘快捷键 + + <方向键>来在他们之间切换移动。假设工作区内的该窗口没有在IDE上按照网格吸附，那么在你按下可行的8个方向的方向键的时候IDE就必须选择你打算要看的窗口，而该选项就允许你设置所使用的“浏览角度”（Look Angle）。默认值为40° ###为方便理解本条目可能需要补一张图###

  • 连接线所连接两个栏目的间隔宽度（The padding between two columns in a chain）：工作区中的元素彼此之间通常被“连锁”起来（比如说，代码编辑器会被连接到事件编辑器，而事件编辑器再被连接到物体编辑器，这样就形成一个工作链（Work Chain）），那么该选项就允许你设置任意两个相连窗口的之间的水平距离。默认值为30px（30像素）。

  • 工作链的行距The padding between two rows in a chain：如前所述，工作区元素彼此通常都是被“连锁”起来的，该选项允许你设置任意两个相连窗口的纵向距离。默认值为20px（20像素）。

  • 连接线需要渲染成多少段？（How Many Segments Should Render Chains Have?）：此处你可以设置渲染连接线所使用的最大段数。降低该值可能有助于在低配置机器上发挥更好的性能。默认值为20.

  • 工作区中连接线的颜色（The link colour for workspace chains）：该颜色被用于显示工作区元素之间的连接线。默认颜色为$99DD00，透明度为153（$99）

  • 使用快捷键在窗口间移动时，我们是否应该重置回1:1（Whether we should reset back to 1:1 when moving between windows using shortcuts）：如前所述，你可以在工作区中通过使用键盘快捷键在窗口之间跳转。你也可以使用快捷键 + 鼠标滚轮 缩放工作区。设置该选项将意味着如果你的工作区被缩放了之后又要使用快捷键跳转窗口，则工作区会自动退回到1:1的缩放视图，并且居中所选窗口。该选项默认为关。

  • 允许工作区的连接线交叉（Allow workspace chains to overlap）：启用该选项后将允许工作区内的连接线彼此交叉重叠。该选项默认为关

  • 选择使用的IDE皮肤（Select the IDE skin to use）：此处你可以选择IDE所能使用的各种皮肤。默认地GameMaker Studio 2本身自带两种皮肤，一个是浅色皮肤，一个是深色皮肤，初始安装时会使用深色皮肤。

  • DPI改写（DPI Override）：该选项允许用你自己的分辨率设定去覆盖默认分辨率设定。你必须先启用该选项然后去设置它的值（默认值为96），并且一旦你应用了修改，你就会被提示去重启IDE，而这就可能要求你保存当前打开的工程的所有工作。
