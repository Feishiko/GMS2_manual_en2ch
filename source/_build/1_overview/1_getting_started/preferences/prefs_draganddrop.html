偏好设置 - 拖拽按钮（Drag and Drop）


拖拽按钮设置用于定义拖拽按钮脚本编辑器看起来是什么样子。下面列举了各种可用选项及每一个的描述。

  • 默认状态下工具箱的位置？（What is the default Toolbox location?）：默认状态下DnD的工具箱（持有各种动作库）会出现在编辑器的右侧，但是你可以使用这个选项来让他出现在左侧。

  • 概览图什么时候出现在节点的旁边？（When does the overview appear next to a node?）：一个节点的“概览”是出现在某个动作工作区中某个动作旁边的一段简短的描述性文字，比如这个销毁实例（Destroy Instance）动作：你可以选择启用一直显示、启用它们在动作块折叠时候显示（默认）或者是禁止它们显示。

  • 转换拖拽按钮为GML时默认该消息（Default message for converting Drag and Drop to GML）：当你酱拖拽按钮事件或脚本转换为GML代码时，默认地你会得到一个关于你要做什么并且询问你是否继续的警告消息。将该选项置为"是"（OK）就会不显示这条警告信息而直接进行变动。

  • 工具箱中动作按钮的缩略图最大尺寸（Maximum size of toolbox Action thumbnails）：在动作按钮工具箱的底部你会看到一个滑条来改变拖拽按钮的图标大小。这项设定将被用于定义允许的最大尺寸（默认为64像素）。

  • 工具箱中动作按钮的缩略图最小尺寸（Minimum size of toolbox Action thumbnails）：在动作按钮工具箱的底部你会看到一个滑条来改变拖拽按钮的图标大小。这项设定将被用于定义允许的最小尺寸（默认为16像素。）

  • 工具箱中动作按钮缩略图的默认尺寸（Default size of toolbox Action thumbnails）：这个设置用于定义工具箱中所有动作图标的默认大小。（默认为32像素）

  • 最近使用动作的最大数量（Maximum amount of recently used actions）：在动作工具箱的顶端有一个库，其持有你最近使用过的动作按钮。你可以使用这个选项来改变最近使用的动作的最大显示数量（默认为5）。

  • 按住鼠标到拖动节点所需的毫秒时间（Time in milliseconds to hold the mouse down to drag nodes）：当你点击一个节点并拖拽时，你就会在动作代码中改变节点的位置。然而如果按住鼠标一段时间This setting sets the number of milliseconds to have to wait with the mouse button pressed before the node can be moved (not repositioned). Default is 350.

  • Should new toolboxes collapse it you have a floating toolbox?: This option (which is on by default) will collapse the toolbox dock in any new editors that are open if you have "pulled" a toolbox out of an event to become a floating window. Switching this off will mean that all events will open with the toolbox dock open regardless od any floating windows.

  • Should the overview panel be collapsed by default?: Here you can choose to hide the overview panel of the Action Code Editor. By default this is disabled and enabling it will remove the plain text list of action chains from the editor.
