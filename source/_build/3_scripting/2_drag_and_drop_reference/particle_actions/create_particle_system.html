Create Particle System


描述

This action will initialise the particle system ready for you to create and use particle types and emitters with it. You can set the render depth of the system, much the same as you can set the depth of different layers within a room, where a low draw depth means that it will appear on top of all things drawn with a higher depth (so a depth for the system of, for example, -1000 would draw over all layers with a depth greater than that), and a high draw depth placing it below everything with a lower draw depth (a depth for the system of 1000 would draw under all layers with a depth less than than that).

You also need to supply a target variable that will hold the unique particle system ID value. This ID value is what will be used in other functions to set properties of the system or to tell GameMaker Studio 2 what system to use when creating particles. You can choose to flag this variable as being a temporary local variable, but that will mean that it is discarded at the end of the action script or event, meaning that the system will still exist but leave you without any means to access it, so care must be taken when using this (in general systems will never need to be stored in a temporary local variable, but for certain things, like drawing particles to a surface, it may be required).

NOTE: A particle system, once created, remains resident in the system memory for the duration of the game, and so if you need to recreate it at any point (for example, if you call the Restart Game action), then you must destroy the current system otherwise you get a memory leak. A memory leak happens when a resource is added to memory and then "de-referenced", meaning that GameMaker Studio 2 has no way to interact with that resource any more. This leads to the memory being clogged by inaccessible resources which in turn can cause issues with your game. To destroy a particle system when no longer required use the action Destroy Particle System.
